# 复刻 files.gallery 全功能到 r2-gallery — 实施计划

## 一、功能差距分析

### files.gallery 核心功能 vs r2-gallery 已实现

| 功能 | files.gallery | r2-gallery | 差距 |
|------|--------------|------------|------|
| **文件浏览** | 6种布局 (list/imagelist/blocks/grid/rows/columns) | 2种 (Grid/List) | 缺4种布局 |
| **排序** | name/kind/size/date/shuffle + 方向 | name/size/mtime + asc/desc | 缺kind排序、shuffle |
| **文件过滤** | files_include/exclude 正则 + UI过滤 | 无 | 完全缺失 |
| **目录过滤** | dirs_include/exclude 正则 | 无 | 完全缺失 |
| **侧边栏菜单** | 递归目录树、最大深度、排序、缓存、localStorage状态 | 基础目录树 | 缺缓存、深度控制、状态持久化 |
| **主题** | 3主题 (contrast/dark/light) + 系统偏好 | dark/light + 系统偏好 | 缺contrast主题 |
| **布局切换** | 下拉菜单切换6种布局 | Header按钮切换 | 可用但需扩展 |

| **文件管理** | 重命名/移动/复制/删除/新建/重复 | 重命名/移动/删除/新建 | 缺复制、重复 |
| **分享链接** | URL快捷方式 + .url文件 | 后端有、前端无UI | 缺前端UI |
| **EXIF信息** | IPTC + EXIF提取 + 显示 | EXIF提取 + 显示 | 需增强IPTC |

## 三、技术架构决策

### 3.1 架构选择: Cloudflare Workers + R2 + D1

**优势**:
- 全球边缘部署，低延迟
- R2无出口费用，适合大文件存储
- D1 SQLite兼容，开发简单
- Workers Assets直接服务前端

**限制**:
- 无FFmpeg/ImageMagick (需WASM替代)
- Worker内存限制 (128MB)
- 无持久连接 (不适合实时协作)
- 执行时间限制 (30秒)

### 3.2 缩略图方案

**当前**: @jsquash/* WASM (JPEG/PNG -> WebP)
**建议**:
- 保持WASM方案作为默认
- 添加GIF/WebP/BMP支持
- 视频缩略图: 使用Cloudflare Stream或预生成

### 3.3 播放器方案

**推荐**: Plyr.js (files.gallery同款)
- 轻量级 (约40KB)
- 支持HLS
- 可定制UI
- MIT开源

### 3.4 编辑器方案

**推荐**: CodeMirror 6
- 模块化设计
- Markdown支持
- 语法高亮
- 轻量级

### 3.5 ZIP方案

**推荐**: JSZip (前端)
- 纯前端实现
- 无需服务器支持
- 适合Cloudflare Workers环境
